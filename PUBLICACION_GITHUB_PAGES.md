# Publicación En GitHub Pages

Este proyecto está preparado como sitio estático. No necesita `npm`, build ni servidor especial.

## Pasos

1. Crear un repositorio público en GitHub, por ejemplo `pulso-ambiental-mty`.
2. Subir estos archivos a la raíz del repositorio:
   - `index.html`
   - `styles.css`
   - `app.js`
   - `metro9-data.js`
   - `metro9.geojson`
   - `README.md`
   - `.nojekyll`
3. Entrar a `Settings > Pages`.
4. En `Build and deployment`, seleccionar:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Esperar a que GitHub genere el enlace público.
6. Registrar el enlace en el documento final y en la evidencia del proyecto.

## URL Final

El sitio quedó publicado y validado en:

`https://heronrisk.github.io/pulso-ambiental-mty/`

## Nota De Presupuesto

GitHub Pages no agrega costo directo para un repositorio público. El presupuesto del proyecto puede mantener el hosting como `USD 0` y conservar como gasto directo únicamente el uso de ChatGPT o herramientas de IA.
